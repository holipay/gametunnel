# Security Fixes Plan

## Fix 1: Remove CORS wildcard from status API

**File**: `internal/server/status.go`

**Problem**: `Access-Control-Allow-Origin: *` on lines 125, 149, 171 allows any website to read token-protected server status via JavaScript.

**Change**: Remove the three `w.Header().Set("Access-Control-Allow-Origin", "*")` lines from `handleStatusJSON`, `handleMetricsJSON`, and `handleRoomsJSON`. The status page HTML is served from the same origin, so CORS is not needed. The JSON API should only be accessed from the same origin or via server-side tools.

---

## Fix 2: Fix `t.conn` data race in client tunnel

**File**: `internal/client/tunnel.go`

**Problem**: `writeUDP` (line 494-503) reads `t.conn` without synchronization. On reconnect, `Connect()` closes the old conn (line 185) and assigns a new one (line 203) while the old `sendLoop` goroutine may still be reading `t.conn` during its drain phase.

**Change**: Pass `conn` as a parameter to `sendLoop` so each goroutine captures a local copy:
1. Change `sendLoop` signature: `func (t *Tunnel) sendLoop(ctx context.Context, conn *net.UDPConn)`
2. Change `writeUDP` signature: `func (t *Tunnel) writeUDP(conn *net.UDPConn, data []byte, addr *net.UDPAddr)`
3. Update call site at line 252: `t.sendLoop(runCtx, t.conn)`
4. Update all `t.writeUDP(...)` calls inside `sendLoop` to `writeUDP(conn, ...)` (or `t.writeUDP(conn, ...)`)
5. This eliminates the unsynchronized read of `t.conn` ŌĆö each `sendLoop` goroutine works with its own captured conn reference.

## Fix 3: Restrict config file permissions

**File**: `internal/client/config.go`

**Problem**: Line 147 writes config with mode `0644` (world-readable). Config may contain `RoomPassword`.

**Change**: Change `0644` to `0600` on line 147.

---

## Fix 4: Handle DNS resolution error in register

**File**: `internal/client/register.go`

**Problem**: Line 204 silently ignores `net.ResolveUDPAddr` error ŌĆö if resolution fails, `clientAddr` is nil, weakening HMAC address binding.

**Change**: Log the error and fall back gracefully (don't fail registration, since the server may have sent a valid address):
```go
clientAddr, err = net.ResolveUDPAddr("udp", acp.ClientAddr)
if err != nil {
    log.Printf("resolve client addr: %v", err)
}
```
This is a defense-in-depth improvement ŌĆö the server-side `ClientAddr` should always be valid, but we shouldn't silently ignore resolution failures.

---

## Fix 5: Panic on nonce fallback instead of using predictable seed

**File**: `internal/crypto/crypto.go`

**Problem**: Line 95 uses `time.Now().UnixNano()` as nonce seed when both `rand.Int` and `rand.Read` fail. Predictable nonce in ChaCha20-Poly1305 is catastrophic (key recovery).

**Change**: Replace the time-based fallback with `panic`:
```go
panic(fmt.Sprintf("crypto: cannot generate random nonce: %v", err))
```
If the system's CSPRNG is truly broken, the program should not silently continue with weak encryption.
