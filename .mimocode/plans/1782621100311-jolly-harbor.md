# Bug Fix Plan ‚Äî GameTunnel Codebase Audit

## Critical & High Bugs (7 issues)

### 1. CRITICAL ‚Äî LZ4 encoder/decoder length mismatch
**File:** `internal/netutil/lz4.go:157-160` (encoder) vs `:254-262` (decoder)

Encoder subtracts 15 per extension byte; decoder adds 255. Any literal run >15 bytes decompresses to garbage.
**Fix:** Align encoder extension loop with LZ4 spec ‚Äî subtract 255 per 0xFF byte emitted.

### 2. HIGH ‚Äî FEC recovery completely broken (3 sub-issues)
- `internal/netutil/fec.go:179` ‚Äî `processDataPacket` is unexported, so receiver can never feed data packets to the decoder
- `internal/client/recv.go:72-87` ‚Äî FEC packets are raw bytes, but `DecodeLenient` is called first; FEC check at line 84 is unreachable
- `internal/netutil/fec.go:276-293` ‚Äî cleanupLoop deletes ALL groups every 5s regardless of age; no timestamp on `fecGroup`

**Fix:** Export `ProcessDataPacket`; move FEC check before protocol decode in `receiveFromServer`; add `created time.Time` to `fecGroup` and only expire aged groups.

### 3. HIGH ‚Äî Crypto replay window race
**File:** `internal/crypto/crypto.go:193-206`

`checkReplayWindow` new-highest path does non-atomic read-modify-write on `highestCounter` + `replayBitmap`. Concurrent `Decrypt` calls can corrupt the bitmap, enabling replay.
**Fix:** Wrap the advance-window path in a CAS loop or use a mutex.

### 4. HIGH ‚Äî Concurrent ReadFromUDP in tryRebind
**File:** `internal/client/keepalive.go:271-276` + `recv.go:44`

`tryRebind` reads from `t.conn` directly while `receiveFromServer` also reads from the same conn. RebindAck can be consumed by the wrong goroutine; `SetReadDeadline` affects both.
**Fix:** Use a dedicated channel or response buffer pattern instead of raw ReadFromUDP in tryRebind.

### 5. HIGH ‚Äî Room.Stop() double-close panic
**File:** `internal/server/room.go:238` + `server.go:429,832`

`Close()` and `keepaliveLoop` can both call `Stop()`, closing the `r.done` channel twice.
**Fix:** Use `sync.Once` for `Stop()`.

### 6. HIGH ‚Äî Nil TUN factory panic
**File:** `internal/client/tunnel.go:372`

`t.newTUNFunc` can be nil if factory was never set.
**Fix:** Guard with nil check before calling.

### 7. HIGH ‚Äî t.serverAddr assigned before error check
**File:** `internal/client/tunnel.go:224-234`

`t.serverAddr` is set before checking if `ListenUDP` succeeded.
**Fix:** Move assignment after error check.

## Medium Bugs (6 issues)

### 8. DecodeLenient accepts corrupted unencrypted packets
**File:** `internal/protocol/protocol.go:136-145`

CRC failure falls back to no-check decode for all packets, not just encrypted.
**Fix:** Remove `DecodeLenient`; use `DecodeChecked` for unencrypted and `DecodeSkipCRC` for encrypted paths explicitly.

### 9. isNewFormat heuristic fragility
**File:** `internal/protocol/messages_data.go:126-128`

IPv6 payloads or future flag values can misclassify format.
**Fix:** Add version negotiation or use a more robust discriminator.

### 10. LZ4 decompression match-copy bounds check
**File:** `internal/netutil/lz4.go:300-305`

`matchStart+i` not checked against `dstLen`.
**Fix:** Add bounds check on the read index.

### 11. ipConnCount leak on NAT rebind auth
**File:** `internal/server/room_auth.go:322-343`

Successful auth after NAT rebind doesn't adjust `ipConnCount`.
**Fix:** Decrement old IP count, increment new IP count.

### 12. Stale addrToRoom cleanup TOCTOU race
**File:** `internal/server/server.go:780-816`

Snapshot-then-delete pattern can remove valid newly-created entries.
**Fix:** Re-check room.addrMap under write lock before deleting.

### 13. Data race on peer.PublicAddr
**File:** `internal/client/recv.go:191-201`

`peer.PublicAddr` read after RLock released.
**Fix:** Snapshot `PublicAddr` while holding the lock.
